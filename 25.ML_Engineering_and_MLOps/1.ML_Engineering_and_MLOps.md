# Chapter 25: ML Engineering and MLOps

## Why this chapter matters
A model is useful only when it is reliable in production.
This chapter connects modeling to deployment, monitoring, and maintenance.

## Learning goals
1. Convert notebook work to reproducible pipelines.
2. Define model/data versioning strategy.
3. Build monitoring and alerting plan.
4. Run safe deployment strategies.

## Step 1: Project structure standard
- `data/` immutable snapshots
- `features/` feature logic
- `models/` training + inference code
- `pipelines/` orchestration jobs
- `tests/` unit + data contract tests

## Step 2: Artifact/version policy
Track and store:
- dataset hash
- feature pipeline version
- model weights version
- metrics report

## Step 3: Monitoring in production
Three layers:
1. service metrics: latency, error rate, throughput
2. data metrics: nulls, ranges, category shifts
3. model metrics: AUC/MAE drift, calibration drift

## Step 4: Deployment strategy
1. shadow mode
2. canary release
3. progressive traffic increase
4. rollback automation

## Step 5: Governance
- model cards
- bias/fairness audit
- PII policy and access controls
- retrain approval criteria

## Assignment
Take one existing Kaggle practice project and write:
1. production inference API contract
2. monitoring dashboard spec
3. retraining trigger policy
4. rollback runbook

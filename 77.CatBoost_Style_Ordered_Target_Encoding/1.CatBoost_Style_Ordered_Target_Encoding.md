# Chapter 77: CatBoost-Style Ordered Target Encoding (Pure Python)

## Why this chapter matters
High-cardinality categorical features are common in industry data: user IDs, merchant IDs, campaigns, SKUs, geohashes. Kaggle pipelines often fail because target encoding leaks labels and inflates CV scores.

CatBoost introduced **ordered statistics** to avoid this leakage while preserving signal.

## Learning goals
1. Implement leaky and leakage-safe target encodings.
2. Build ordered target statistics in strict time/order sequence.
3. Train logistic regression on encoded features in pure Python.
4. Compare train/validation gap across encoding methods.
5. Visualize leakage signatures and encoding stability.

## Core idea
If we encode category `c` using mean target over all rows containing `c`, each row sees its own label. That is leakage.

Ordered encoding fixes this:
- process data in order
- for row `i`, encode category `c_i` using only rows `< i`
- update category stats after computing current row's encoding

With smoothing:

`enc_i = (sum_prev[c_i] + alpha * global_mean) / (count_prev[c_i] + alpha)`

## Kaggle + production implications
- ordered or OOF encoding is mandatory for robust CV.
- leak-prone features create overconfident, brittle models.
- production drift often appears first in rare-category buckets.

# Chapter 79: Kaggle Adversarial Validation and Drift-Aware Feature Selection

## Why this chapter matters
One of the most common Kaggle failures is overfitting to training distribution quirks that do not match the leaderboard/test distribution. Adversarial validation is a practical way to detect this mismatch before final submission.

## Learning goals
1. Build adversarial validation classifier (`train` vs `test`).
2. Rank features by distribution-shift importance.
3. Remove unstable features and retrain target model.
4. Compare in-domain validation vs out-of-domain performance.
5. Visualize drift severity and robustness tradeoff.

## Industry context
- Growth, pricing, fraud, and recommendation systems often face temporal/domain shift.
- Offline validation can be over-optimistic if data split ignores shift.
- Drift-aware feature selection improves deployment reliability.

## Core workflow
1. Train baseline target model on all features.
2. Build adversarial dataset:
- label source train as 0
- label source future/test as 1
3. Train adversarial classifier.
4. Use adversarial feature weights as drift signal.
5. Remove top drift features and retrain target model.
6. Compare robustness (OOD AUC/logloss).

## Practical warning
Do not remove every shifted feature blindly. Some shifted features still carry stable signal. Use iterative ablation and monitor both validation and OOD metrics.

## Assignment
1. Add monotonic constraints manually to stable features.
2. Add temporal split instead of random split and compare.
3. Use per-feature PSI with adversarial weights for joint ranking.
4. Build robust ensemble: all features + stable-only model blend.

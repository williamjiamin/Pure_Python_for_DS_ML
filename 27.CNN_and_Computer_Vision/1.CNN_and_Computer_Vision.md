# Chapter 27: CNN and Computer Vision

## Why this chapter matters
Computer vision production systems depend on strong data pipelines, augmentation policy, transfer learning, and evaluation by failure slices.

## Learning goals
1. Understand convolution and receptive fields.
2. Build a basic CNN pipeline.
3. Use transfer learning workflow.
4. Perform robust evaluation (precision/recall by class and slice).

## Step 1: Convolution intuition
A kernel scans local neighborhoods and learns spatial detectors (edges, textures, shapes).
Depth increases semantic abstraction.

## Step 2: CNN baseline architecture
- Conv -> ReLU -> Conv -> ReLU -> Pool
- repeat blocks
- global average pooling
- dense classifier head

## Step 3: Transfer learning standard process
1. start with pretrained backbone
2. freeze backbone, train head
3. unfreeze upper blocks and fine-tune with lower LR
4. evaluate calibration and confusion matrix

## Step 4: Augmentation strategy
Use label-preserving transforms:
- random crop/resize
- horizontal flip
- color jitter
- cutout/mixup (task dependent)

## Step 5: Production concerns
- input quality gate (blurry, too dark)
- class imbalance handling
- latency-aware model compression (quantization/distillation)

## Assignment
Train a small image classifier and produce:
1. confusion matrix
2. top failure classes
3. improvement plan by data collection or augmentation
